# 灵魂回声 Soul Echo

一个沉浸式 AI「灵魂按摩」网页应用。你写下此刻想说的一句话，系统感应你的意图与情绪，邀请五位历史灵魂之一——**苏轼 / 王阳明 / 佛陀 / 尼采 / 庄子**——在专属的视听场域里与你展开一段有节奏的对话，最后凝结成一张可下载、可分享的「回声卡」。

它不是通用聊天机器人，也不是心理咨询，而是一次「被一位有分量的灵魂温柔接住」的相遇。

## 特性

- **软路由**：一句话进入。系统按语义把你引向合适的灵魂，而不是让你先挑角色。困境、请教、报喜、闲聊，各有各的接法。
- **五种思想指纹**：每位灵魂是一套「本体 + 姿态」的角色宪法——接失意时是雨夜同坐的人，见证喜事时是举杯的老友。
- **场域仪式**：约 2.5 秒的入场过渡、逐行浮现的对话、郑重登场的回声时刻。轻、可跳过、尊重 `prefers-reduced-motion`。
- **回声卡**：竖版 1080×1440 的图片作品——微缩场景简笔画 + 竖排回声 + 朱红「回声」印 + 纸纹，纯 Canvas 绘制，可下载。
- **安全护栏**：危机语句在首页与对话中途都会被接住，转为求助文案，不继续角色扮演。
- **离线可玩**：不配置任何模型 key 也能走通完整体验（关键词路由 + 内置文案兜底）。

## 快速开始

需要 **两个终端**（前端把 `/api` 代理到本地接口服务）。

```bash
npm install
```

终端 1 —— 接口服务（端口 4173）：

```bash
npm run api
```

终端 2 —— 前端（端口 5173）：

```bash
npm run dev -- --host 127.0.0.1
```

浏览器打开 http://127.0.0.1:5173/ 。

### 无模型 key 也能完整体验

不配置任何 key，即可跑通：**首页输入 → 感应推荐 → 入场 → 对话 → 回声 → 保存卡 → 卡墙**。无 key 时走关键词路由与各灵魂内置的兜底文案。

### 接入真实模型（可选）

配置环境变量后，感应与对话改由模型生成（OpenAI 兼容接口，亦识别 DeepSeek）：

```bash
AI_API_KEY=sk-... AI_BASE_URL=https://api.deepseek.com AI_MODEL=deepseek-chat npm run api
```

支持 `AI_API_KEY` / `AI_BASE_URL` / `AI_MODEL`，以及 `OPENAI_*`、`DEEPSEEK_*` 作为回退。请勿把 key 提交进仓库。

## 技术栈

- 前端：React 18 + TypeScript + Vite + Tailwind CSS
- 接口：Node 原生 `http` 服务（`server.mjs`），无框架
- 回声卡：Canvas 2D 绘制，导出 PNG，无额外依赖
- 角色系统：`src/shared/souls.mjs` —— 每位灵魂是一个配置对象（路由、场景、角色宪法、兜底文案）

## 构建
