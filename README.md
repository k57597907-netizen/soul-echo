# 灵魂回声 Soul Echo

一个沉浸式 AI 情感陪伴网页应用。

用户只需写下一句此刻想说的话，系统便会感应其来意与情绪，从 **苏轼、王阳明、佛陀、尼采、庄子** 五位思想人格中推荐适合的回应者，邀请用户进入专属的视听场域，完成一段有节奏的对话，并把最终获得的精神回应凝结成一张可以保存和分享的「回声卡」。

它不是通用聊天机器人，也不是心理咨询，而是一次「被一位有分量的灵魂温柔接住」的相遇。

## 产品理念

很多时候，人并不缺少信息，而是缺少一种恰当的回应：

- 难过时，不想马上听到解决方案；
- 内耗时，需要有人帮助自己迈出最小一步；
- 焦虑时，需要暂时松开控制；
- 报喜时，希望这件事被郑重地看见；
- 想不通时，希望借助另一种思想视角重新理解；
- 说不清需求时，也不想先研究应该选择哪个 AI 角色。

「灵魂回声」不追求回答所有问题，而是在用户情绪最真实的一刻，用合适的人格、语气与节奏回应他。

## 核心体验

```text
说出一句话
→ 感应用户来意
→ 推荐三位灵魂
→ 选择并进入专属场域
→ 完成有节奏的对话
→ 生成和保存回声卡
```

### 一句话进入

首页不要求用户预先选择功能、角色或情绪标签。开心的、难过的、想不通的，都可以直接说出来。

例如：

- “努力了很久，还是没有结果。”
- “我考上了，三年了终于考上了。”
- “最近一直拖延，我知道该做什么，但就是动不了。”
- “尼采怎么看自卑？”
- “今晚想和苏轼喝杯茶。”
- “朋友失恋了，我想替她求一句话。”

单次首页输入最多 500 字。建议一次聚焦一件事，使用自己最自然的语言，不必组织成完整问题。

### 意图感应

系统会先判断用户带着什么来意进入：

- **倾诉 `pour_out`**：正在经历困境或负面情绪；
- **请教 `inquire`**：想从某种思想视角理解问题；
- **相见 `meet`**：点名想见某位灵魂；
- **见证 `witness`**：分享好消息或人生节点；
- **赠言 `gift`**：替别人求一句话；
- **抉择 `crossroad`**：处在人生选择或十字路口；
- **闲聊 `casual`**：没有明确目标，只想聊一聊；
- **危机 `crisis`**：出现明确的自伤或自杀表达。

有模型 Key 时，系统使用模型判断意图、情绪形态与推荐人物；没有模型 Key 时，则通过本地关键词和人物路由规则完成感应。

### 灵魂推荐

系统通常会给出三位候选者。每张推荐卡包含人物名称、回应片段、推荐原因和对应思想场域。

系统负责降低选择成本，但不会替用户作唯一决定。用户可以选择最能触动自己的回应，也可以返回修改原来的那句话。

## 五位灵魂

| 灵魂 | 更适合的处境 | 回应风格 | 不太适合的处境 |
| --- | --- | --- | --- |
| **苏轼** | 失意、落差、努力无果、成败执念 | 温润松弛、有烟火气，像雨夜同坐的朋友 | 需要立即制定强执行方案 |
| **王阳明** | 内耗、拖延、犹豫、知道却做不到 | 清醒坚定、短句、有行动感 | 只想安静承接悲伤与丧失 |
| **佛陀** | 执念、失去、离别、情感痛苦 | 安静慈悲、留白、不催促 | 需要被激励去竞争或争取 |
| **尼采** | 虚无、自卑、麻木、无力感 | 锋利而有热度，唤醒但不羞辱 | 情绪非常脆弱、只想被安静接住 |
| **庄子** | 焦虑、比较、控制欲、过度用力 | 轻盈旁观、略带幽默 | 需要明确执行步骤的具体决策 |

每位灵魂都不是简单套用名言，而是拥有一套完整的角色配置，包括核心信念、思想指纹、称呼方式、意象、表达禁区、思考动作、对话策略、场景设计与无模型时的兜底回应。

## 三种主要对话方式

### 倾诉与困境

适用于失意、焦虑、内耗、失恋、自我怀疑或努力没有结果等时刻。

对话通常按以下节奏展开：

1. **镜像共情**：先理解用户正在经历什么，不急于讲道理；
2. **视角重构**：通过人物独有的思想方式，帮助用户换一个角度理解当下；
3. **破壁动作**：邀请用户完成一个足够小、可以立即执行的动作；
4. **结缘回声**：把本次相遇凝结成一句属于用户的精神回应。

破壁动作可能是喝一口水、做三次缓慢呼吸、整理桌面一角、写下下一步的第一行，或者暂时松开对某件事的追逐。

### 喜悦与人生节点

适用于考试通过、找到工作、完成长期目标、毕业、生日、结婚、走出低谷或任何值得纪念的进步。

对话节奏为：

```text
接过喜悦 → 放大意义 → 凝结贺辞
```

人物在这里不是安慰者，而是一位有分量的见证者。他会郑重地看见用户的努力，与用户一同为这件事高兴，并留下带有本次经历影子的贺辞。

### 相见与闲聊

适用于“今晚想和苏轼喝杯茶”“我想见见庄子”或没有明确议题的自由交谈。

这类对话不会强行套入情绪疏导节奏，而是保持自然交流。如果用户在聊天中逐渐转向倾诉或分享喜悦，人物会顺着语境调整回应方式。

## 使用说明

### 1. 说出此刻最想说的话

在首页输入一句真实的话，点击「感应」。不必先选择人物，也不必把它写成标准问题。

### 2. 选择一位灵魂

阅读系统推荐的回应片段和推荐理由，选择此刻最愿意与之交谈的人物，点击「进入场域」。

### 3. 完成入场

人物场景会逐渐亮起，名称、入场语和开场回应依次出现。入场过程约 2.5 秒，可以点击跳过，并支持系统的 `prefers-reduced-motion` 设置。

### 4. 继续对话

直接回复人物的问题或说出新的感受。系统会根据来意、对话轮次与人物思想方式组织后续回应，并在适合的时候邀请用户完成一个小动作。

### 5. 生成回声

当用户至少完成两次发言后，可以生成回声。系统会记录本次相遇的人物、最初输入、意图、情绪形态、回应摘要和最终回声。

### 6. 查看“我的回声”

生成后的内容会进入本地回声卡墙。当前浏览器最多保存最近 30 张卡片，不需要注册账号。

### 7. 保存为图片

回声卡可以导出为 1080 × 1440 的竖版 PNG 图片，包含：

- 人物专属微缩场景；
- 竖排回声文字；
- 朱红色「回声」印章；
- 人物与日期信息；
- 可选的用户原话。

移动端可以长按预览图保存，桌面端可以点击「下载回声卡」。

## 应用场景

### 努力很久却没有结果

> 我准备了很久，但最后还是没选上，我开始怀疑是不是自己不行。

系统可以帮助用户把“这一次没有结果”和“我整个人失败了”区分开，承认失落、松动成败执念，并留下一句可以反复回看的回应。

### 拖延与自我内耗

> 我明明知道该做什么，但每天都在拖延，越拖越讨厌自己。

系统会把注意力从抽象的自我审判转向当下能完成的最小动作，让行动先于情绪恢复。

### 失恋、离别与失去

> 我知道这段关系已经结束了，但还是放不下。

用户可以在不被催促“赶紧放下”的前提下承认失去，重新理解纪念、执念与自我照顾之间的关系。

### 自卑、虚无与力量感缺失

> 我感觉自己什么都做不好，也不知道自己有什么价值。

系统可以帮助用户从“我是不是没有价值”转向“我愿意创造什么”，用有力量但不羞辱的语言唤醒行动意愿。

### 分享好消息

> 我考上了，三年了终于考上了。

系统会进入见证模式，让长期努力被认真看见，放大这件事对用户的意义，并生成一句属于这一时刻的贺辞。

### 人生选择与十字路口

> 我不知道该留在稳定的工作里，还是去做真正想做的事。

人物不会直接替用户决定，而是帮助用户识别自己真正看重和害怕失去的东西，换一种尺度理解选择，并找到下一步可以验证的行动。

### 替朋友求一句话

> 朋友最近失恋了，我想替她求一句话。

用户可以获得比“别难过”“加油”更贴近情境的表达，并把回声卡保存下来发送给朋友。

### 睡前整理情绪

在一天结束时，说出最放不下的一件事，用几分钟完成一次精神整理，留下一句第二天仍愿意看到的话。

### AI 产品与交互设计展示

项目也适合用于展示：

- 多角色 Prompt 架构；
- 意图路由与情绪识别；
- 对话节奏状态机；
- React + Node 的生成式 AI 应用实现；
- 人文思想与 AI 的融合设计；
- Canvas 生成式视觉卡片。

## 无模型 Key 也能使用

不配置任何模型 Key，也可以走通完整流程：

```text
首页输入 → 感应推荐 → 入场 → 对话 → 生成回声 → 保存卡片
```

此时系统使用本地关键词匹配、人物路由规则和内置回应，不产生模型 API 费用。

## 接入真实模型

配置 OpenAI 兼容接口后，感应与对话会由模型动态生成。

支持：

```text
AI_API_KEY
AI_BASE_URL
AI_MODEL
```

同时兼容 `OPENAI_*` 和 `DEEPSEEK_*` 环境变量。

DeepSeek 示例：

```bash
AI_API_KEY=sk-... \
AI_BASE_URL=https://api.deepseek.com \
AI_MODEL=deepseek-chat \
npm run api
```

OpenAI 示例：

```bash
OPENAI_API_KEY=sk-... \
OPENAI_MODEL=gpt-4o-mini \
npm run api
```

请只在服务端设置 API Key，不要把密钥写入前端代码或提交到仓库。

## 本地运行

### 环境要求

- Node.js
- npm
- 现代浏览器

### 安装依赖

```bash
npm install
```

### 启动接口服务

终端 1：

```bash
npm run api
```

接口默认运行在 `http://127.0.0.1:4173`。

### 启动前端

终端 2：

```bash
npm run dev -- --host 127.0.0.1
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

Vite 会将前端的 `/api` 请求代理到本地接口服务。

### 构建

```bash
npm run build
```

## 数据与安全

### 本地保存

回声卡和静音设置保存在当前浏览器的 `localStorage` 中：

- 不需要注册账号；
- 不会自动同步到其他设备；
- 清除浏览器站点数据后，本地记录也会被清除；
- 配置真实模型后，对话内容会发送给所配置的模型服务商处理。

### 危机表达

首页输入与对话过程中都设置了危机语句检查。当检测到明确的自伤或自杀表达时，系统会停止人物角色扮演，提示用户立即联系真实的人、当地紧急电话或心理援助热线。

「灵魂回声」提供的是情绪表达和思想陪伴体验，不能替代专业心理咨询、医疗服务或紧急救助。

## 技术架构

- **前端**：React 18 + TypeScript + Vite + Tailwind CSS
- **接口服务**：Node.js 原生 `http` 服务
- **模型协议**：OpenAI 兼容的 `/chat/completions` 接口
- **角色系统**：配置化人格、路由、场景、姿态与兜底文案
- **回声卡**：Canvas 2D 绘制并导出 PNG
- **本地数据**：浏览器 `localStorage`
